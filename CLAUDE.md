# Websiteerstellung

## Tech Stack

    Next.js 15 + TypeScript + Tailwind CSS
    Framer Motion für Animationen


## Design-Regeln

    Nutze das AskUserQuestion Tool, um den Nutzer über das Websitedesign zu interviewen, damit du die Vorstellungen des Nutzers genau abbilden kannst
    Nutze den frontend-design Skill für alle UI-Entscheidungen
    Nutze UI/UX Pro Max für Design-System-Generierung
    Nutze audit-website für SEO usw.
    Nutze ggf. 21st.dev für Component-Inspiration (falls vorgegeben)
    Nutze web-design-guidlines für compliance mit Web interface Guidelines für Vercel
    Nutze ggf. skill-creator um meinen wünschen gerecht zu werden und für eine optimale Umsetzung, falls notwendig
    Keine generischen AI-Aesthetics
    Bold, distinctive Design-Choices
    Performance-optimiert (Core Web Vitals)

    # Ziel
Baue ein DACH Affiliate Discovery Portal für kuriose, lustige und kaufstarke Produkte.

# Prioritäten
1. Live gehen
2. SEO-fähige Struktur
3. Schnelle Ladezeiten
4. Manuelle Produktpflege zuerst
5. Automatisierung später

# Verboten
- Keine generischen AI-Layouts
- Keine automatische Veröffentlichung ohne Review
- Keine Änderungen am Design-System ohne Begründung
- Keine überkomplizierte Architektur im MVP
